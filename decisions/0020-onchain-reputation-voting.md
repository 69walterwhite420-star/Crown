# ADR 0020 — Репутация в кости: ончейн-счётчики и трастлесс-голосование споров (G3b, вариант Б)

Дата: 2026-07-02. Статус: принято владельцем, спека написана (`docs/onchain-voting-spec.md`),
реализация не начата.

## Контекст

Спор — последний большой кусок кости, живущий в коже: голоса/подсчёт на сервере, исход на цепь
переносит один ключ (bounded-резолвер, D1/D2). На нём держатся принятые компромиссы CR-1/2/3.
G3b в ROADMAP обещал «ончейн commit-reveal голосование». Ключевой вопрос дизайна: **откуда
контракт узнаёт ВЕС голоса**, если репутация — оффчейн-свёртка журнала.

## Решение (выбор владельца, 2026-07-02)

**Вариант Б — репутация материализуется ончейн**: счётчик-PDA на пару «канал × кошелёк»,
инкрементится в той же транзакции, что несёт донат; чекпоинты дают вес «на момент открытия
спора»; голос — бесплатная подпись сообщения (ed25519, тот же примитив, что SIWS/аттестация),
подача пачками permissionless с возмещением из банка спора; конверты (commit-reveal); депозит
оспаривающего; финализация по таймеру кем угодно. Резолвер-ключ удаляется, в эскроу его место
занимает PDA программы споров.

Ключевые производные решения:

1. **Слоты голосов в счётчике** вместо аккаунта-на-голос: предельная стоимость голоса по
   хранению = 0 (иначе 100k голосов = десятки тысяч долларов возвратной, но замороженной ренты).
2. **Наказание только за активный проигрыш**: −50/сгоревший депозит — только если комьюнити
   явно проголосовало «выполнил»; no-quorum/ничья → без штрафа, депозит назад (наследует
   текущую оффчейн-логику repEffects).
3. **Арбитражные издержки платит проигравший** (из банка спора); податели и финализатор
   возмещаются по факту с потолком → рынок доставки, наш сервис не обязателен.
4. **Параметры голосования фиксируются заранее** (ончейн-конфиг канала + фиксация на момент
   создания задания или таймлок изменений) — стример не может поднять порог голоса перед спором.
5. **Дельты репутации без движения денег (+10/−50) — легальны в кости**: счётчик пишут только
   протокольные события (платёж, проверенный цепочкой; вердикт, посчитанный контрактом);
   новые писатели — только апгрейдом через таймлок.
6. **Число/правила/смысл разделены**: число не меняет никто; правила — апгрейд под
   мультисиг+таймлок (сжигание — после годовой стабильности); смысл (тиры/пороги) — кожа,
   свободно.

## Отвергнутые альтернативы

- **Вариант А (merkle-снэпшоты весов через пруф-якорь)** — дешевле строить, но вес остаётся
  «доверенно-проверяемым» (публикатор снэпшота = мы). Владелец выбрал полную кость. А остаётся
  как понятный fallback, если Б упрётся в непредвиденное.
- **Голос транзакцией с fee-payer (заправщик)** — бесплатно для голосующего, но голос проходит
  через чьего-то плательщика в момент подачи (мягкий рычаг цензуры) и требует SOL-спонсора
  онлайн. Подпись сообщения строго лучше: портативна, подаётся кем угодно и когда угодно.
- **«Додеп»: платное расширение мест в споре с наградой репутацией** — отвергнуто владельцем
  после разбора: (а) репутация за возвратный залог/деньги мимо стримера ломает proof-of-spend;
  (б) искусственный дефицит мест + платное расширение = рычаг манипуляции явкой; (в) споры
  должны оставаться судом, не тотализатором (§14 спеки игры). Сохранена безопасная часть:
  спонсорство банка спора с возвратом и косметическим признанием (кожа), без репутации.
- **Множители/настраиваемый курс репутации** — подтверждён отказ (ADR 0007): жёсткий курс
  1 USDC = 1 очко — то, что делает ончейн-счётчик тривиальным (`+= сумма`); настраиваемый курс
  потребовал бы ончейн-версионирования конфигов курса.

## Последствия

- Донат-путь меняется: голые SPL-переводы → инструкция программы (CPI-переводы + инкремент
  счётчика). Некастодиальность сохраняется (программа денег не держит, подписывает донор);
  кошелёк показывает симуляцию сумм. Это самое чувствительное изменение — аудит первым делом.
- Умирают: резолвер (D1/D2), CR-1, CR-3; CR-2 сужается до UI-видимости. Кнопки резолвера и
  `ESCROW_RESOLVER` удаляются.
- Редеплой эскроу-программы (RESOLVER → PDA программы споров) совмещается с mainnet-пинами
  ESC-7/ESC-9.
- upgrade authority повышается в статусе: из «страховки» в «орган эволюции правил» — мультисиг
  + таймлок на мейннете обязательны ещё сильнее.
- Ревизор `verify-export` получает проверку «журнал == счётчики».
- Нагрузочная репетиция 10k–100k голосов на devnet — обязательная фаза V3 до мейннета.

## Фазы

V0 (счётчики + ончейн-конфиг) → V1 (споры, открытое голосование, без резолвера) → V2 (конверты,
депозиты, банк) → V3 (нагрузка) → V4 (аудит + mainnet-гейты). Светофоры владельца — спека §13.
