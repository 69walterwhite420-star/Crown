# ADR 0011 — Батч хардненга по аудиту: SIWS-binding, finalized, failMode, bigint

- **Статус:** принято
- **Дата:** 2026-06-20
- **Контекст:** после C1 (ADR 0008), C2 (ADR 0009) и H3 (ADR 0010) остались мелкие M/L-находки аудита
  «как украсть на mainnet». Закрываем их одним проходом — каждая правка маленькая и независимая.

## Решения

### M1 — SIWS: domain/issued-at/expires-at + ограничение сторов
Подписываемое сообщение связано с **доменом** и **временем** (в духе CAIP-122): пользователь видит, куда
входит, а подпись для одного приложения/окна не переносится на другое. Сервер хранит `issuedAt` в записи
nonce и строит сообщение **байт-в-байт** одинаково при выдаче (`issueNonce`) и проверке
(`verifyAndIssueToken`) — клиент по-прежнему подписывает то, что вернул сервер, поэтому правка серверная.
Карты `nonces`/`sessions` теперь ограничены (`MAX_NONCES`/`MAX_SESSIONS`) и чистятся: протухшие удаляются,
при переполнении вытесняются старейшие — `__authNonce` неаутентифицирован, иначе это безграничный
memory-DoS.

### M2 — зачёт по `finalized` в chain-режиме
`ingest` (путь зачёта репутации) запрашивает транзакцию с commitment **`finalized`** в chain-режиме —
защита от реорга на mainnet (откатившаяся tx не зачтётся). На devnet остаётся `confirmed` для отзывчивости.
Управляется общим серверным `CHAIN_MODE`.

### L1 — `failMode` только вне прода
Инъекция ошибок (`failMode`, dev-тулинг для прогона error-стейтов) honor-ится только при `!IS_PROD`. Раньше
конкурентный клиент мог прислать `failMode:true` и перетереть глобальный флаг singleton-стора, роняя
читающие методы у других пользователей (гриферство).

### L2 — лимит длины `__bigint` в кодеке
`codec.decode` конвертирует `{__bigint}` в BigInt только при `/^-?\d{1,40}$/` (деньги в micro-USDC влезают
с огромным запасом). Гигантская строка больше **не** скармливается `BigInt()` — это была DoS-нагрузка на
публичный RPC. Нестрогий тег проходит как обычный объект, без бросков.

### DRY — `src/server/runtime.ts`
Общий серверный модуль с `IS_PROD` и `CHAIN_MODE` (не NEXT_PUBLIC) — единая формула chain-режима для
C1-гейта (`route`) и M2-зачёта (`ingest`), чтобы не дублировать.

## Проверено
- **SIWS-раунд реальным ed25519-ключом** (tsx + сервер): `__authNonce` (сообщение содержит
  domain/issued-at/expires-at) → подпись → `__authVerify` → токен → `getSession` отдаёт наш адрес;
  **реплей** погашенного nonce и **поддельная подпись** отбиты.
- **L2** (tsx): нормальные bigint round-trip; 40 цифр конвертятся; 100k-значный `__bigint` НЕ
  конвертится за 0.05ms (BigInt не вызван, нет зависания); мусор `1e999` не конвертится.
- C1-гейт жив после рефактора `CHAIN_MODE`→runtime; `tsc` чист; prettier применён. (Тест-файлы временные,
  в коммит не входят.)

## Честные ограничения / открытое
- **L3** — токен в `localStorage` (XSS-компромисс). Перевод на httpOnly-cookie + CSRF — отдельный больший шаг.
- **M3** — нет фонового индексер-сервиса; проглоченная ошибка `ingestSignature` означает, что честный донор
  может не получить зачёт (надёжность, не безопасность).
- Открыты **H1** (payout диктуется сервером — нужен дизайн ончейн-привязки) и **H2** (сбор за активацию).
- `APP_DOMAIN` — серверная env; на проде задать реальный домен приложения.
