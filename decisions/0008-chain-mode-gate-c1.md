# ADR 0008 — Серверный гейт chain-режима: запрет оффчейн-симуляции доната (C1)

- **Статус:** принято
- **Дата:** 2026-06-20
- **Контекст:** аудит «как украсть, пока оно на mainnet» (после ADR 0006) нашёл критический пролом **C1**.
  Оффчейн-симуляция `createDonation` (`MockDataProvider`, оставлена для api/mock-тулинга без кошелька)
  осталась в белом списке RPC и достижима в **любом** деплое, включая chain/mainnet. Сервер не знает про
  `NEXT_PUBLIC_DATA_SOURCE` — это **клиентская** переменная, `route.ts` её не читает; режим «chain» живёт
  только в браузере. Поэтому любой вошедший по SIWS кошелёк (вход бесплатный, gasless) мог послать
  `POST {method:"createDonation", ...}` и получить **полностью зачтённый донат + репутацию + текст на
  оверлей БЕЗ единого перевода USDC**. Последствия (воспроизведено `curl`):
  - подделка продукта — репутация есть продукт; форж бесплатно и без лимита;
  - инъекция в оверлей — `auto_if_clean` + `CLEAR` → `SHOWN` → текст на стриме как «оплаченный» (показ
    без оплаты — обратное нарушение инварианта §4.7 «деньги ≠ показ»);
  - слом §4.4 — независимый пересчёт из цепочки НЕ воспроизведёт форжённые события (у них нет ончейн-
    основы) → юридический замок «любой может пересчитать» рушится.

  ADR 0006 закрыл спуф личности и авторизацию мутаций, но не трогал то, что сама симуляция обходит
  ончейн-модель истины. Это её отдельный фикс.

## Решения

### 1. Серверный флаг `CHAIN_MODE` (не `NEXT_PUBLIC`)
Новая **серверная** env `CHAIN_MODE` — недоступна клиенту, не подделать из браузера. Определение
(`src/app/api/v1/rpc/route.ts`):

```ts
const CHAIN_MODE = process.env.CHAIN_MODE === "on" || (IS_PROD && process.env.CHAIN_MODE !== "off");
const CHAIN_FORBIDDEN = new Set<string>(["createDonation"]);
```

В chain-режиме методы из `CHAIN_FORBIDDEN` (сейчас — `createDonation`) отдают ошибку `CHAIN_MODE` (403).
Гейт стоит **до** авторизации и диспатча, поэтому не зависит от наличия токена. Начисление репутации в
chain-режиме делает **только** `ingestSignature` — трастлесс-приём ончейн-доната (`server/ingest.ts` сам
достаёт tx из цепочки, сверяет пару 97/3 + memo и payout-ATA канала).

### 2. Fail-safe по умолчанию (mainnet не «переоткрывает» дыру)
В `production` симуляция закрыта **по умолчанию**, без явной конфигурации — забытый env на mainnet не
вернёт пролом (в отличие от fail-open дефолтов, ср. C2). Escape-hatch `CHAIN_MODE=off` — для редкого
prod-деплоя api/mock-режима (демо без кошелька). В dev (`NODE_ENV != production`) по умолчанию открыто —
mock/api-тулинг (createDonation-симуляция для прогона экранов) продолжает работать. Управляющий сигнал —
**явный** `CHAIN_MODE`, а не `NODE_ENV` (NODE_ENV — лишь fail-safe-бэкстоп).

### 3. Побочно: `ingestSignature` завёрнут в try/catch
Публичный неаутентифицированный эндпоинт ронялся в **HTTP 500** на кривой/неизвестной подписи (web3.js
бросал `SolanaJSONRPCError`) — в отличие от общего диспатча, эта спец-ветка не ловила ошибку. Теперь
отдаёт чистый `ok:false` с кодом ошибки. Мелкая, но реальная DoS-шероховатость публичного входа.

## Проверено / не проверено

**Проверено (dev-стенд `next dev`, `curl`):**
- `CHAIN_MODE=on`: `createDonation` → `CHAIN_MODE` 403; `listChannels` → ok (гейт хирургический);
  `ingestSignature` на фейк-подписи → чистый `ok:false` (не 500), т.е. ончейн-путь гейтом не задет;
- dev без `CHAIN_MODE`: `createDonation` доходит до хендлера (`NO_CHANNEL`) — тулинг цел;
- логическая таблица `CHAIN_MODE` по 4 кейсам (dev/prod × on/off/unset); `tsc --noEmit` чисто; prettier применён.

**НЕ проверено:** прод-build (`next start`) c реальным SIWS-токеном — гейт срабатывает ДО авторизации,
поэтому от токена не зависит (SIWS-раунд покрыт ADR 0006).

## Честные ограничения / открытое из аудита
Это закрывает подделку через RPC-симуляцию, но не остальные находки аудита 2026-06-20:
- **C2** — дефолтный `TREASURY_OWNER`/`OPERATOR_ADDRESS` = закоммиченный devnet-адрес с ключом в
  плейнтекст-файле; fail-open на mainnet при незаданном env. → следующий фикс (fail-closed в проде).
- **H1** — `payoutAddress` диктуется сервером, кошелёк донора не может его независимо сверить
  (компрометация сервера → редирект будущих донатов).
- **H2** — `activateChannel` флипает статус без сбора (анти-флуд-экономика отсутствует).
- **H3** — личность запроса живёт на общем singleton-сторе; безопасно только пока методы стора синхронны
  (мина перед миграцией на Postgres — реальный `await` даст межзапросную эскалацию).
- **M/L** — SIWS без domain/expiry-binding + неаутентифицированный безлимитный nonce; ingest на
  `confirmed`, не `finalized`; проглоченная ошибка ingest без фонового индексера; лимит bigint; токен в
  localStorage.

При добавлении новых оффчейн-симуляций — дополнять `CHAIN_FORBIDDEN`.
